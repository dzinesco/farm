## Project Overview

FRMX is a browser-based design tool for residential prefab wall panels. The core model is a **layered wall assembly** system — one canonical model produces plan views, 3D views, elevation views, and BOM/reports.

**Key domain concepts**: Wall → Panel → LayerStack + FramingModel. A Panel is a fabrication unit carrying the full layer stack, framing layout, and edge/connection metadata.

## Architecture

### Monorepo Structure

```
frmx/
├── apps/web/                 # Next.js 15 (App Router) + Tailwind CSS v4
└── packages/
    ├── calculator/           # ✅ imperial feet-inches arithmetic, fractions, roof pitch
    ├── core/                 # ✅ types, validation, IDs (nanoid), Zod schemas
    ├── geometry/             # ✅ 2D/3D primitives, wall-local frames, polyline utilities
    ├── framing-engine/       # ✅ studs (full/jack/king/cripple), headers, cripples
    ├── panelizer/            # ✅ wall → panels at max-length and opening boundaries
    ├── layers/               # ✅ layer stack presets (2x6 exterior, interior partition)
    ├── modules/              # ✅ opening presets, corner/intersection configs, components
    ├── materials/            # ✅ BOM rollup, waste factors, lumber/sheathing catalogs
    ├── reports/              # ✅ panel schedule, cut lists, lumber + sheathing summaries
    └── project-io/           # ✅ IndexedDB persistence, .frmx.json export/import
```

### Technology Stack

- **App**: Next.js 15 (App Router), React 19, Tailwind CSS v4, Zustand for state
- **State**: Zustand store with history/undo-redo, selection (wall + panel), view modes
- **2D views**: Canvas 2D with custom dimension rendering (pan/zoom, hit testing, grid)
- **3D**: react-three-fiber + drei (configured but deferred — needs React 19 compat layer)
- **Persistence**: IndexedDB via `@frmx/project-io` + `.frmx.json` export/import
- **Styling**: Tailwind v4 with `@tailwindcss/postcss` (NOT `tailwindcss` as PostCSS plugin directly)
- **Imperial math**: `@frmx/calculator` — all dimension math flows through this package

### Domain Model

```
Project
├── Standards (spacing, lumber grades, max panel size, layer presets)
├── MaterialsCatalog
├── ModuleLibrary
│   ├── OpeningPreset[]     # window bay, door jamb (with RO dimensions)
│   ├── CornerPreset[]       # L-wrap, two-panel + corner stud pack
│   ├── IntersectionPreset[] # L, T, cross connections
│   └── Component[]          # chase, electrical pack, hold-downs
└── Building
    └── Level[]
        ├── Wall[]            # centerline/chain, height, type
        │   ├── Opening[]
        │   └── Panel[]       # ordered along wall
        │       ├── LayerStack[]   # ordered layers, thickness, material ref
        │       ├── FramingModel   # studs, headers, plates, cripples, blocking
        │       ├── ModuleInstance[]
        │       └── Connections[]
        └── Space[] (optional)
```

**FramingModel** is generated by panelizer + framing-engine, then editable in elevation (move stud, add blocking). Overrides are stored as deltas so regeneration can warn on conflicts. All changes flow through a command stack enabling full undo/redo.

### Prefab-Specific Rules

- Stud spacing: 16" / 24" OC
- Plate count: single/double top, single bottom
- Max panel dimensions for transport (configurable per plant, default 96")
- Break at openings with configurable offset
- Corner panels: L-wrap vs two-panel + corner stud pack
- Wall intersections: L, T, and cross connections with preset configurations

### Core Design Principle

**Manual control is paramount.** Any auto-generated result (framing layout, panel splits, module placements) must be editable afterward. Overrides are stored as deltas on top of the generated model so regeneration can warn on conflicts. The user always retains final say — automation assists, never dictates.

### Units

Imperial default (feet-inches). The `calculator` package handles all imperial arithmetic — fractions (1/4", 1/8", 1/16"), feet-inches strings ("10'-6 3/4"), roof pitch (rise/run ↔ degrees ↔ percentage), linear/area/volume totals, and metric conversion. This is the foundation all other packages depend on for dimension math.

```bash
npx turbo build

# Development (run from apps/web)
cd apps/web && pnpm dev

# Build web app
cd apps/web && pnpm build
```

## Performance Targets

60fps orbit with instanced studs; 2D pan/zoom < 16ms frame on M1 laptop. Target: ~80 walls, ~400 panels per building.

## Build Notes

- **Tailwind v4**: Use `@tailwindcss/postcss` in `postcss.config.js`, NOT `tailwindcss` directly as a plugin
- **tsup external**: Packages with `workspace:*` dependencies must list them as `external` in `tsup.config.ts` to avoid bundling issues
- **ESLint**: Disabled in `.eslintrc.js` — re-enable with proper ESLint 9 flat config when needed
- **3D view**: react-three-fiber needs additional React 19 compatibility configuration — currently a placeholder
- **Calculator tests**: 44 passing — run with `cd packages/calculator && pnpm test`
