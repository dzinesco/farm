FRMX v1 — Refined plan
Product positioning
FRMX is a browser-based design tool for residential prefab wall panels: designers lay out plans, the system panelizes walls into shop units, each panel is a layered assembly (framing, insulation, sheathing, finishes), and plan / 3D / elevation views plus BOM and shop outputs all come from one model.

Primary user: designer/estimator at a prefab plant (office browser).
Secondary (later): shop tablet (PWA, read-only + label print).

Panel = fabrication unit on a wall run: carries full layer stack, framing layout derived from rules, and edge / connection metadata (splices, lifts, crane points). (manual control and placement is a must)

**Core design principle: manual overrides always win.** Auto-generated results (framing, panel splits, module placements) are editable afterward. Overrides stored as deltas so regen can warn on conflict. User always retains final say.

        │   ├── Opening[]
        │   └── Panel[]     # ordered along wall
        │       ├── LayerStack[]   # ordered layers, thickness, material ref
        │       ├── FramingModel   # generated + manual overrides
        │       ├── ModuleInstance[]
        │       └── Connections[]  # to adjacent panels / structure
        └── Space[] (optional, for area reports)
FramingModel is generated by panelizer + framing-engine, then editable in elevation (move stud, add blocking). Overrides are stored as deltas on top of generated layout so regen can warn on conflict.

3D = extrusion/sweep of each layer + instanced studs; 2D plan = wall footprints + panel seam lines; elevation = wall-local 2D with layer toggles and framing symbols.

Performance target (web): one building, ~80 walls, ~400 panels, full layers — 60fps orbit with instanced studs; 2D pan/zoom < 16ms frame on M1 laptop.

Panelizer + framing (prefab rules v1)
Inputs: wall geometry, height, type, openings, Standards, layer preset.
Outputs: Panel[] each with FramingModel + LayerStack clone from wall type.

Rules (MVP set):

Split wall into panels ≤ maxPanelLength (transport)
No panel < minPanelWidth unless flagged manual
Prefer breaks away from opening RO by breakOffset
Exterior: full-height panels; headers as generated framing, not separate “panel” unless user splits
Interior partition: may allow stacked panels if height > plant table limit
Regenerate policy: “Apply rules” shows diff (seams moved, stud count delta); user accepts or keeps manual overrides.

Reports (residential prefab v1 ship set)

Weight estimate: density × volume per layer (configurable).
